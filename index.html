Personal-use notice: This app is a private automation used only by me (RJ) to post updates to my own LinkedIn account from my own GitHub repositories. It is not offered as a public service, and no other users are supported.

What this app does

The app runs as an automation (typically via GitHub Actions) when I push commits to my repositories, generates a short summary, and publishes that summary to my LinkedIn account.

Information processed

The app may temporarily process the following:

  • GitHub activity data: repository name, branch, commit IDs, commit messages, changed file paths, and compare links.
  • Optional code context: limited diff/patch text or line-change counts, if enabled for better summaries.
  • LinkedIn data: authorization tokens and my LinkedIn member identifier (URN) required to publish a post.

How information is used

Information is used only to (1) decide whether a push is “postable,” (2) generate a short post draft, and (3) publish the post to my LinkedIn account. The app is designed to avoid posting sensitive data (for example, strings that look like API keys/tokens).

AI / third-party processing (optional)

If AI summarization is enabled, the app sends the minimum necessary GitHub activity details (such as commit messages and changed file paths, and optionally small diff snippets) to a text-generation service to produce the post text. No unrelated personal data is intentionally sent.

Data storage and retention

The app does not use a dedicated database. It is intended to process data in memory and discard it after posting. However, some data may be retained by third parties as part of normal operations:

  • GitHub: repository history and commit data (as part of Git itself), and GitHub Actions workflow logs/artifacts if enabled.
  • LinkedIn: the published post and associated metadata, according to LinkedIn’s platform behavior.
  • Hosting/automation provider: standard security/operational logs (for example, GitHub Actions logs).

Secrets (such as LinkedIn OAuth credentials/tokens) are stored as private CI secrets (for example, GitHub Actions Secrets) and are not published.

Sharing

Data is not sold. Data is shared only with services necessary to run the automation: GitHub (to run workflows and access repo content), LinkedIn (to publish posts), and optionally an AI text-generation provider (if enabled).

Your choices / control

  • Disable the GitHub Actions workflow (or equivalent automation) to stop processing.
  • Revoke the app’s access in LinkedIn account settings to stop posting.
  • Rotate/revoke tokens and update CI secrets at any time.

Changes to this policy

If the app’s behavior changes in a way that affects privacy (for example, storing additional data or adding new third-party services), this policy will be updated and the effective date will be changed.
